题目传送门

前置知识

后缀表达式如果不明白，可以参考这篇博客。在这里，我们做简短介绍：

后缀表达式，又称逆波兰式，指的是不包含括号，运算符放在两个运算对象的后面，所有的计算按运算符出现的顺序，严格从左向右进行(不再考虑运算符的优先规则)。例如：

四则表达式（我们生活中的算术式）：4+6×7−104+6\times 7-104+6×7−10
后缀表达式：444 666 777 ×\times× +++ 101010 −-−

思路

首先肯定考虑贪心，把所有数排序，把大数加上，小数减去，非常简单！代码如下（我这个是用优先队列的大根堆，也能达到效果，分析也在代码中）：

源代码 - C++
复制

#include<bits/stdc++.h>
using namespace std;
int n,m,s;
priority_queue<int,vector<int>,less<int> >pq;
int main(){
	cin>>n>>m;
	for(int i=1;i<=n+m+1;i++)cin>>s,pq.push(s);//加入堆
	long longans=0;
	n++;//加比减多一个
	while(n--){
		//cout<<pq.top()<<endl;调试
		ans+=pq.top();//加
		pq.pop();
	}
	while(m--){
		//cout<<pq.top()<<endl;调试
		ans-=pq.top();//减
		pq.pop();
	}
	cout<<ans;//output
	return 0;
}

我们来想一下原因：后缀表达式顺序如同栈，先进先出，所以先算出的也有可能参与后面的算式，这在四则表达式中类同括号的作用。所以，最后的结果受括号影响，受减号和加号的顺序影响，举个例子就能看出：

而我们的输出就应该是 131313 而不是 555，我们还可以发现在后缀表达式中真正会被减的数的个数可以被我们降到一（当减号都在一起时），所以我们只要牺牲一个最小的值，把其他数加起来就好啦。 详细的实现可以参考代码：

代码

源代码 - C++
复制

#include<bits/stdc++.h>
using namespace std;
int n,m,s,maxx=-1,minn=1<<30;
long long ans=0;
int main() {
	cin>>n>>m;//input
	if(m==0) {//m==0，无最小值可牺牲的特判
		for(int i=1; i<=n+m+1; i++)cin>>s,ans+=s;//全加
		cout<<ans;
		return 0;
	}
	for(int i=1; i<=n+m+1; i++) {
		cin>>s;
		ans+=abs(s);
		maxx=max(maxx,s);
		minn=min(minn,s);//求最小值
	}
	cout<<(maxx-minn)+ans-abs(maxx)-abs(minn);//print，牺牲最小值
	return 0;
}

时间复杂度也十分优秀，连 sort 都不用了。

P8683 [蓝桥杯 2019 省 B] 后缀表达式 题解

题意简述

link

  • 给定 NNN 个加号、 MMM 个减号以及 N+M+1N+M+1N+M+1（下文记为 KKK )个整数 A1,A2,⋯ ,AN+M+1A_1,A_2,\cdots,A_{N+M+1}A1​,A2​,⋯,AN+M+1​

  • 求由这 NNN 个加号、 MMM 个减号以及 KKK 个整数凑出的合法的后缀表达式中，结果最大的是哪一个。

  • 例如输入 1 2 1 2 3，则 2 3 + 1 - 这个后缀表达式结果是 444，是最大的。

（在这里我们将这 KKK 个整数记为 1-idx 的集合 aaa )

题目分析

先上 30pts 代码：

源代码 - C++
复制

ll ans=0;
sort(a+1,a+n+m+1+1);
for(int i=1;i<=m;i++)ans-=a[i];
for(int i=m+1;i<=n+m+1;i++)ans+=a[i];
cout<<ans;

这是一个明显的贪心思路：先排序，加上 n+1n+1n+1 个大数，再减去 mmm 个小数。

但是这个贪心能够明显的举出反例：

源代码
复制

0 4
1 2 3 4 5

答案是 131313, 他会输出 −5-5−5。因为这是一个后缀表达式，转换到中缀表达式时可能会带着括号！

现在我们的首要目标就是把括号的影响忽略掉。观察上面的例子可以发现，答案 13=5−((1−2)−3)−4)=5+4+3+2−113=5-((1-2)-3)-4)=5+4+3+2-113=5−((1−2)−3)−4)=5+4+3+2−1。让我们在举几个例子看看：

  • 输入为 0 2 1 2 3,输出为 5=3−(1−2)=3+2−15=3-(1-2)=3+2-15=3−(1−2)=3+2−1;
  • 输入为 1 2 4 3 2 1,输出为 8=4+3−(1−2)=4+3+2−18=4+3-(1-2)=4+3+2-18=4+3−(1−2)=4+3+2−1;
  • 输入为 1 2 4 3 -2 -1,输出为 10=4+3−(−2)−(−1)=4+3+1−(−2)10=4+3-(-2)-(-1)=4+3+1-(-2)10=4+3−(−2)−(−1)=4+3+1−(−2);

可以看出，

  • 在 M=0M=0M=0 时答案为 ∑i=1Kai\sum_{i=1}^K a_i∑i=1K​ai​，
  • 而在 M>0M>0M>0 时答案为 ∑i=2K∣ai∣−a1\sum_{i=2}^K \lvert {a_i} \rvert-a_1∑i=2K​∣ai​∣−a1​（此时 aaa 数组已经按从小到大排序）

证明：

首先在 M=0M=0M=0 时答案一定为所有数的和。

在 M>0M>0M>0 时，想让答案最大，那么肯定要多加上大数，减去小数。先把最大的拿出来，放到首项（K−1K-1K−1 个符号，KKK 个数，那么首项一定是正的），把最小的拿出来，放到末项。此时就构成了一个这样的表达式：an±...−(a1±...)a_n\pm...-(a_1\pm...)an​±...−(a1​±...)。

这时其他的数就即可以放到括号外面，又可以放到括号里面。如果是正数括号里就添-,括号外就添+,负数相反。就这样，我们把所有任意个加号和减号都转变为了绝对值！虽然“牺牲”了最小的值，但是仍然换来了最大的收益。

代码

源代码 - C++
复制

#include<iostream>
#include<algorithm>
#define ll long long
using namespace std;
ll a[1000010];int n,m;
ll ans=0;
int main(){
	ios::sync_with_stdio(false);
	cin>>n>>m;
	for(int i=1;i<=n+m+1;i++)
		cin>>a[i];
	sort(a+1,a+n+m+1+1);
	if(m==0)
		for(int i=1;i<=n+m+1;i++)
            ans+=a[i];
	else{
		ans=a[n+m+1]-a[1];
		for(int i=2;i<=n+m;i++)
            ans+=abs(a[i]);//abs是c++的库函数 取绝对值
	}
	cout<<ans;
	return 0;
}

思路（详细！）

如果 MMM 是 000 则没有减号，全是加号，全加起来即可。如果 MMM 大于 000 我们从后缀表达式特征入手：以二叉树后序遍历顺序描绘，则很明显有下面的特征。当我们将新的数放到左子树上且新的符号为负数时，会将原有的数据全部取反。因此，对于 MMM 个负号，我们可以构造出 111 到 MMM 个负号，而如果在负号的过程中加入了加法，那么相当于括号外是减法，括号内是加法，去括号后，加法也能变成减法，从而增加了负号的数量，因为有 NNN 个加号，故而可以增加 000 到 NNN 个负号，所以 NNN 个加号和 MMM 个减号可以构成 111 到 N+MN+MN+M 个负号因为有 N+M+1N+M+1N+M+1 个数，而可以构造的负号数范围在 111 到 N+MN+MN+M，所以至少会减一个数，至少会加一个数，为了使结果最大，我们应该减去最小的数，加上最大的数，也就是将序列排序后的序列两端，减去左端，加上右端。

而序列中间的数，依据这个特征，我们的数据负号实际上可以自由更改加减，又因为题目要求求最大值，则我们只需要根据以上反转的特性，合理构造负号，利用一个负数将多个负数都尽可能转换为正值，也即将序列中负数前面的符号构造为负号，以负负得正，正数前面就是加号，最后的效果其实就是序列中间的数全部取绝对值。

从而我们就能得到最优代码。

代码

源代码 - C++
复制

#include <bits/stdc++.h>
using namespace std;
const int N = 2e5 + 10;

long long cnt;
int main() {
	int a[N];
	int n, m;
	cin >> n >> m;
	for (int i = 0; i < n + m + 1; i++)
		cin >> a[i];
	if (m == 0) {
		for (int i = 0; i < n + m + 1; i++)
			cnt += a[i];
	} else {
		stable_sort(a, a + n + m + 1);
		cnt -= a[0];
		cnt += a[n + m];
		for (int i = 1; i < n + m; i++)
			cnt += abs(a[i]);
	}
	cout << cnt << endl;
	return 0;
}

1. 编程思路。

题目给出了后缀表达式中加号的个数为 nnn，减号的个数为 mmm。在输入数据 aia_iai​ 时进行预处理，统计出所有正数的个数（设为 ppp）、正数的和（设为 psumpsumpsum）和最小的正数（设为 pminpminpmin），同时也统计出所有负数的个数（设为 qqq）、所有负数的和（设为 nsumnsumnsum）和最大的负数（设为 nmaxnmaxnmax）。

要使后缀表达式的值最大，分以下四种情况讨论。

1）m=0m=0m=0，没有减号，只能将全部数加起来，因此表达式的值为 psum+nsumpsum+nsumpsum+nsum。

2）有减号，但全部的数均为正数，即 m≠0 and p=n+m+1m\ne 0 \ and \ p=n+m+1m=0 and p=n+m+1。此时可以这样利用减号。

如果只有 111 个减号（m=1m=1m=1），很简单，减去最小的正数即可，答案为 psum−2×pminpsum-2\times pminpsum−2×pmin。

如果有多个减号，我们可以让最小的正数连续减去 m−1m-1m−1 个正数，最后用一个减号来减去这个差。形式化的表示如下：

a1+a2+...+an+1−(pmin−an+2−an+3−...−an+m)a_1+a_2+...+a_{n+1}-(pmin-a_{n+2}-a_{n+3}-...-a_{n+m})a1​+a2​+...+an+1​−(pmin−an+2​−an+3​−...−an+m​)

=a1+a2+...+an+1+an+2+an+3+...+an+m−pmin=a_1+a_2+...+a_{n+1}+a_{n+2}+a_{n+3}+...+a_{n+m}-pmin=a1​+a2​+...+an+1​+an+2​+an+3​+...+an+m​−pmin

=psum−2×pmin=psum-2\times pmin=psum−2×pmin

3）有减号，但全部的数均为负数，即 m≠0 and q=n+m+1m\ne 0 \ and \ q=n+m+1m=0 and q=n+m+1。此时可以这样利用减号。

如果只有 111 个减号，同样简单，用最大的负数（绝对值最小）减去其他所有负数的和，这样相当其他所有负数和的绝对值减去一个绝对值最小的负数，表达式的值肯定最大，为 −nsum+2×nmax-nsum+2\times nmax−nsum+2×nmax。

如果有多个减号，我们可以让最大的负数连续减去 m−1m-1m−1 个负数，最后用一个减号来减去其他 n+1n+1n+1 个负数的和。形式化的表示如下：

nmax−a1−a2−...−am−1−(am+am+1+...+an+m)nmax-a_1-a_2-...-a_{m-1}-(a_{m}+a_{m+1}+...+a_{n+m})nmax−a1​−a2​−...−am−1​−(am​+am+1​+...+an+m​)

=nmax−(a1+a2+...+am−1+am+am+1+...+an+m)=nmax-(a_1+a_2+...+a_{m-1}+a_{m}+a_{m+1}+...+a_{n+m})=nmax−(a1​+a2​+...+am−1​+am​+am+1​+...+an+m​)

=−nsum+2×nmax=-nsum+2\times nmax=−nsum+2×nmax

4）有减号，并且也有正数，也有负数。此时处理起来简单。用减号减去负数即可。

如果负数个数比减号多，可以采用 −(ai+aj) -(a_i+a_j)−(ai​+aj​) 的形式处理，其中 aia_iai​ 与 aja_jaj​ 均为负数；

如果负数的个数比减号少，可以采用 −(ai−aj) -(a_i-a_j)−(ai​−aj​) 的形式处理，其中 aia_iai​ 是负数，aja_jaj​ 是正数。

总之，最后结果一定是所有正数和负数的绝对值相加，后缀表达式的值也最大，为 psum−nsumpsum-nsumpsum−nsum。

2. 源程序。

源代码 - C
复制

#include <stdio.h>
int main()
{
	int n,m,p=0,q=0;
	scanf("%d%d",&n,&m);
	long long psum=0,nsum=0;
	long long pmin=2000000000,nmax=-2000000000;
	int i;
	for (i=0;i<n+m+1;i++)
	{
		long long num;
		scanf("%lld",&num);
		if (num>0ll)
        {
            p++;
            psum+=num;
            if (pmin>num) pmin=num;
        }
		else
        {
            q++;  nsum+=num;
            if (nmax<num) nmax=num;
        }
	}
	if (m==0)       // 负号个数为0
		printf("%lld\n",psum+nsum);
	else
	{
	    if (p==n+m+1)         // 全部为正数
	        printf("%lld\n",psum-2*pmin);
        else if (q==n+m+1)   // 全部为负数
            printf("%lld\n",-nsum+2*nmax);
        else
            printf("%lld\n",psum-nsum);
	}
	return 0;
}

            for(int i=1; i<=n+m+1; i++)cin>>s,ans+=s;//全加
            cout<<ans;
            return 0;
        }
        for(int i=1; i<=n+m+1; i++) {
            cin>>s;
            ans+=abs(s);
            maxx=max(maxx,s);
            minn=min(minn,s);//求最小值
        }
        cout<<(maxx-minn)+ans-abs(maxx)-abs(minn);//print，牺牲最小值
        return 0;
    }

时间复杂度也十分优秀，连 sort 都不用了。

P8683 [蓝桥杯 2019 省 B] 后缀表达式 题解

题意简述

link

  • 给定 NNN 个加号、 MMM 个减号以及 N+M+1N+M+1N+M+1（下文记为 KKK )个整数 A1,A2,⋯ ,AN+M+1A_1,A_2,\cdots,A_{N+M+1}A1​,A2​,⋯,AN+M+1​

  • 求由这 NNN 个加号、 MMM 个减号以及 KKK 个整数凑出的合法的后缀表达式中，结果最大的是哪一个。

  • 例如输入 1 2 1 2 3，则 2 3 + 1 - 这个后缀表达式结果是 444，是最大的。

（在这里我们将这 KKK 个整数记为 1-idx 的集合 aaa )

题目分析

先上 30pts 代码：

源代码 - C++
复制

ll ans=0;
    sort(a+1,a+n+m+1+1);
    for(int i=1;i<=m;i++)ans-=a[i];
    for(int i=m+1;i<=n+m+1;i++)ans+=a[i];
    cout<<ans;

这是一个明显的贪心思路：先排序，加上 n+1n+1n+1 个大数，再减去 mmm 个小数。

但是这个贪心能够明显的举出反例：

源代码
复制

0 4
    1 2 3 4 5

答案是 131313, 他会输出 −5-5−5。因为这是一个后缀表达式，转换到中缀表达式时可能会带着括号！

现在我们的首要目标就是把括号的影响忽略掉。观察上面的例子可以发现，答案 13=5−((1−2)−3)−4)=5+4+3+2−113=5-((1-2)-3)-4)=5+4+3+2-113=5−((1−2)−3)−4)=5+4+3+2−1。让我们在举几个例子看看：

  • 输入为 0 2 1 2 3,输出为 5=3−(1−2)=3+2−15=3-(1-2)=3+2-15=3−(1−2)=3+2−1;
  • 输入为 1 2 4 3 2 1,输出为 8=4+3−(1−2)=4+3+2−18=4+3-(1-2)=4+3+2-18=4+3−(1−2)=4+3+2−1;
  • 输入为 1 2 4 3 -2 -1,输出为 10=4+3−(−2)−(−1)=4+3+1−(−2)10=4+3-(-2)-(-1)=4+3+1-(-2)10=4+3−(−2)−(−1)=4+3+1−(−2);

可以看出，

  • 在 M=0M=0M=0 时答案为 ∑i=1Kai\sum_{i=1}^K a_i∑i=1K​ai​，
  • 而在 M>0M>0M>0 时答案为 ∑i=2K∣ai∣−a1\sum_{i=2}^K \lvert {a_i} \rvert-a_1∑i=2K​∣ai​∣−a1​（此时 aaa 数组已经按从小到大排序）

证明：

首先在 M=0M=0M=0 时答案一定为所有数的和。

在 M>0M>0M>0 时，想让答案最大，那么肯定要多加上大数，减去小数。先把最大的拿出来，放到首项（K−1K-1K−1 个符号，KKK 个数，那么首项一定是正的），把最小的拿出来，放到末项。此时就构成了一个这样的表达式：an±...−(a1±...)a_n\pm...-(a_1\pm...)an​±...−(a1​±...)。

这时其他的数就即可以放到括号外面，又可以放到括号里面。如果是正数括号里就添-,括号外就添+,负数相反。就这样，我们把所有任意个加号和减号都转变为了绝对值！虽然“牺牲”了最小的值，但是仍然换来了最大的收益。

代码

源代码 - C++
复制

#include<iostream>

            cin>>a[i];

        }
        cout<<ans;
        return 0;
    }

思路（详细！）

如果 MMM 是 000 则没有减号，全是加号，全加起来即可。如果 MMM 大于 000 我们从后缀表达式特征入手：以二叉树后序遍历顺序描绘，则很明显有下面的特征。当我们将新的数放到左子树上且新的符号为负数时，会将原有的数据全部取反。因此，对于 MMM 个负号，我们可以构造出 111 到 MMM 个负号，而如果在负号的过程中加入了加法，那么相当于括号外是减法，括号内是加法，去括号后，加法也能变成减法，从而增加了负号的数量，因为有 NNN 个加号，故而可以增加 000 到 NNN 个负号，所以 NNN 个加号和 MMM 个减号可以构成 111 到 N+MN+MN+M 个负号因为有 N+M+1N+M+1N+M+1 个数，而可以构造的负号数范围在 111 到 N+MN+MN+M，所以至少会减一个数，至少会加一个数，为了使结果最大，我们应该减去最小的数，加上最大的数，也就是将序列排序后的序列两端，减去左端，加上右端。

而序列中间的数，依据这个特征，我们的数据负号实际上可以自由更改加减，又因为题目要求求最大值，则我们只需要根据以上反转的特性，合理构造负号，利用一个负数将多个负数都尽可能转换为正值，也即将序列中负数前面的符号构造为负号，以负负得正，正数前面就是加号，最后的效果其实就是序列中间的数全部取绝对值。

从而我们就能得到最优代码。

代码

源代码 - C++
复制

#include <bits/stdc++.h>

            cin >> a[i];
        if (m == 0) {
            for (int i = 0; i < n + m + 1; i++)
                cnt += a[i];
        } else {
            stable_sort(a, a + n + m + 1);
            cnt -= a[0];
            cnt += a[n + m];
            for (int i = 1; i < n + m; i++)
                cnt += abs(a[i]);
        }
        cout << cnt << endl;
        return 0;
    }

1. 编程思路。

题目给出了后缀表达式中加号的个数为 nnn，减号的个数为 mmm。在输入数据 aia_iai​ 时进行预处理，统计出所有正数的个数（设为 ppp）、正数的和（设为 psumpsumpsum）和最小的正数（设为 pminpminpmin），同时也统计出所有负数的个数（设为 qqq）、所有负数的和（设为 nsumnsumnsum）和最大的负数（设为 nmaxnmaxnmax）。

要使后缀表达式的值最大，分以下四种情况讨论。

1）m=0m=0m=0，没有减号，只能将全部数加起来，因此表达式的值为 psum+nsumpsum+nsumpsum+nsum。

2）有减号，但全部的数均为正数，即 m≠0 and p=n+m+1m\ne 0 \ and \ p=n+m+1m=0 and p=n+m+1。此时可以这样利用减号。

如果只有 111 个减号（m=1m=1m=1），很简单，减去最小的正数即可，答案为 psum−2×pminpsum-2\times pminpsum−2×pmin。

如果有多个减号，我们可以让最小的正数连续减去 m−1m-1m−1 个正数，最后用一个减号来减去这个差。形式化的表示如下：

a1+a2+...+an+1−(pmin−an+2−an+3−...−an+m)a_1+a_2+...+a_{n+1}-(pmin-a_{n+2}-a_{n+3}-...-a_{n+m})a1​+a2​+...+an+1​−(pmin−an+2​−an+3​−...−an+m​)

=a1+a2+...+an+1+an+2+an+3+...+an+m−pmin=a_1+a_2+...+a_{n+1}+a_{n+2}+a_{n+3}+...+a_{n+m}-pmin=a1​+a2​+...+an+1​+an+2​+an+3​+...+an+m​−pmin

=psum−2×pmin=psum-2\times pmin=psum−2×pmin

3）有减号，但全部的数均为负数，即 m≠0 and q=n+m+1m\ne 0 \ and \ q=n+m+1m=0 and q=n+m+1。此时可以这样利用减号。

如果只有 111 个减号，同样简单，用最大的负数（绝对值最小）减去其他所有负数的和，这样相当其他所有负数和的绝对值减去一个绝对值最小的负数，表达式的值肯定最大，为 −nsum+2×nmax-nsum+2\times nmax−nsum+2×nmax。

如果有多个减号，我们可以让最大的负数连续减去 m−1m-1m−1 个负数，最后用一个减号来减去其他 n+1n+1n+1 个负数的和。形式化的表示如下：

nmax−a1−a2−...−am−1−(am+am+1+...+an+m)nmax-a_1-a_2-...-a_{m-1}-(a_{m}+a_{m+1}+...+a_{n+m})nmax−a1​−a2​−...−am−1​−(am​+am+1​+...+an+m​)

=nmax−(a1+a2+...+am−1+am+am+1+...+an+m)=nmax-(a_1+a_2+...+a_{m-1}+a_{m}+a_{m+1}+...+a_{n+m})=nmax−(a1​+a2​+...+am−1​+am​+am+1​+...+an+m​)

=−nsum+2×nmax=-nsum+2\times nmax=−nsum+2×nmax

4）有减号，并且也有正数，也有负数。此时处理起来简单。用减号减去负数即可。

如果负数个数比减号多，可以采用 −(ai+aj) -(a_i+a_j)−(ai​+aj​) 的形式处理，其中 aia_iai​ 与 aja_jaj​ 均为负数；

如果负数的个数比减号少，可以采用 −(ai−aj) -(a_i-a_j)−(ai​−aj​) 的形式处理，其中 aia_iai​ 是负数，aja_jaj​ 是正数。

总之，最后结果一定是所有正数和负数的绝对值相加，后缀表达式的值也最大，为 psum−nsumpsum-nsumpsum−nsum。

#include <stdio.h>
    int main()
    {
        int n,m,p=0,q=0;
        scanf("%d%d",&n,&m);

        int i;
        for (i=0;i<n+m+1;i++)
        {
            long long num;
            scanf("%lld",&num);
            if (num>0ll)
            {
                p++;
                psum+=num;
                if (pmin>num) pmin=num;
            }
            else
            {
                q++;  nsum+=num;
                if (nmax<num) nmax=num;
            }
        }
        if (m==0)       // 负号个数为0
            printf("%lld\n",psum+nsum);
        else
        {
            if (p==n+m+1)         // 全部为正数
                printf("%lld\n",psum-2*pmin);
            else if (q==n+m+1)   // 全部为负数
                printf("%lld\n",-nsum+2*nmax);
            else
                printf("%lld\n",psum-nsum);
        }
        return 0;
    }
